## 阶段 1：基础框架（MVP）

### v0.1.0 - 项目脚手架
**目标：** 可运行的基础框架

**任务：**
- [ ] 项目初始化（TypeScript + Node.js）
- [ ] 目录结构搭建
- [ ] 基础 CLI 框架（commander）
- [ ] 配置系统（读取、验证、默认值）
- [ ] 日志系统

**验收标准：**
```bash
$ xca --version
xca v0.1.0

$ xca config init
✅ 配置已创建在 ~/.config/xca/config.json

$ xca config show
当前配置：
  provider: ollama
  model: gemma4:4b
```

### v0.2.0 - Provider 层
**目标：** 支持至少两种 LLM 接入

**任务：**
- [ ] Provider 抽象接口
- [ ] Ollama Provider 实现
- [ ] OpenAI Provider 实现
- [ ] 流式响应支持
- [ ] 配置切换 Provider

**验收标准：**
```bash
$ xca config set provider ollama --model gemma4:4b
✅ 已切换到 Ollama (gemma4:4b)

$ xca config test
🔄 测试连接...
✅ Ollama 连接正常，模型就绪

$ echo "你好" | xca
你好！我是 Xca Agent，有什么可以帮你的吗？
```

### v0.3.0 - 工具系统基础
**目标：** 实现核心工具，支持基础文件操作

**任务：**
- [ ] Tool 抽象接口
- [ ] read_file 工具
- [ ] list_directory 工具
- [ ] write_file 工具
- [ ] 工具调用基础流程

**验收标准：**
```bash
$ xca "读取 README.md"
🔧 使用工具：read_file
参数：{"path": "README.md"}

[Xca Agent 根据文件内容回复...]
```

### v0.4.0 - Agent 核心
**目标：** 完整的对话循环，支持多轮工具调用

**任务：**
- [ ] Agent 编排核心
- [ ] 对话历史管理
- [ ] Token 预算基础管理
- [ ] 交互式 CLI UI
- [ ] 内置命令（/help, /clear, /exit）

**验收标准：**
```bash
$ xca
🤖 Xca Agent v0.4.0
提供商：Ollama (gemma4:4b)
输入 /help 查看帮助

> 创建一个简单的 HTTP server
🔧 使用工具：write_file
...
🔧 使用工具：execute_command (npm init -y)
...
✅ 已创建 server.js

> 运行它
🔧 使用工具：execute_command (node server.js)
Server running on port 3000
```

## 阶段 2：功能完善

### v0.5.0 - 完整工具集
**目标：** 对标 Claude Code 的基础工具能力

**任务：**
- [ ] edit_file（精确编辑）
- [ ] search_files（文本搜索）
- [ ] execute_command（命令执行 + 安全确认）
- [ ] apply_diff（diff 格式应用）
- [ ] 大文件分块读取

**验收标准：**
- 可以完成"帮我把 var 改成 const"这样的任务
- 危险命令有确认提示
- 能处理 1000+ 行的文件

### v0.6.0 - 上下文管理
**目标：** 智能的上下文管理，支持长对话

**任务：**
- [ ] Token 计算和预算管理
- [ ] 对话历史自动摘要
- [ ] 文件上下文缓存
- [ ] 相关文件自动关联
- [ ] /undo 命令

**验收标准：**
- 20+ 轮对话后仍能正常工作
- 大文件不会撑爆上下文
- 能"记得"之前的讨论
